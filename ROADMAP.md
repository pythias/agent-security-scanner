# ai-ass 后续优化规划

## 📊 输出格式增强

### 1. 多格式输出
- [ ] **JSON 输出** (`--format json`) - 便于程序解析和集成
- [ ] **SARIF 输出** - 安全工具通用格式，可集成到 CI/CD
- [ ] **HTML 报告** - 生成美观的网页报告
- [ ] **Markdown 报告** - 生成可分享的 Markdown 报告

### 2. 彩色终端输出
- [ ] 使用 `chalk` / `picocolors` 实现彩色输出
- [ ] 进度条显示扫描进度
- [ ] 表格形式展示结果
- [ ] 动态加载动画

### 3. 交互式模式
- [ ] TUI 界面 (使用 `blessed` 或 `ink`)
- [ ] 选择性扫描 (勾选要扫描的 agents)
- [ ] 结果详情展开/折叠
- [ ] 一键修复建议

---

## 🔍 检测能力增强

### 4. 更多检测器
- [ ] **凭据检测** - 检测更多类型的硬编码凭据 (AWS, GCP, Azure tokens)
- [ ] **恶意代码检测** - 检测挖矿程序、反向 shell
- [ ] **依赖漏洞** - 集成 npm audit / snyk
- [ ] **隐私数据检测** - 检测 PII (身份证、电话、邮箱)

### 5. 智能分析
- [ ] **上下文分析** - 不仅检测模式，还分析代码逻辑
- [ ] **误报过滤** - 过滤示例代码、注释中的误报
- [ ] **严重性智能评估** - 基于利用链评估真实风险

---

## ⚙️ 功能增强

### 6. 配置化
- [ ] 配置文件 (`~/.ai-ass.yaml`) - 自定义检测规则、排除路径
- [ ] 规则自定义 - 用户可添加自己的检测模式
- [ ] 白名单机制 - 忽略已知安全的代码

### 7. 集成能力
- [ ] **GitHub Actions** - 一键集成到 CI/CD
- [ ] **GitLab CI** - MR 安全扫描
- [ ] **Pre-commit Hook** - 提交前自动扫描
- [ ] **VS Code 插件** - 实时安全提示

### 8. 增量扫描
- [ ] 只扫描变更的文件
- [ ] 缓存机制提高速度
- [ ] 并行扫描多目录

---

## 🎨 UI/UX 优化

### 9. 可视化
- [ ] 风险趋势图
- [ ] 按 agent/目录分布饼图
- [ ] 时间线展示历史扫描

### 10. 国际化
- [ ] 中文/英文双语支持
- [ ] 错误信息本地化

## 🚀 性能优化

- [ ] WebAssembly 加速正则匹配
- [ ] 多线程并行扫描
- [ ] 大项目智能分片

## 优先级建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | JSON 输出 | 便于集成 |
| P0 | 彩色输出 | 提升体验 |
| P1 | 配置文件 | 必要定制能力 |
| P1 | HTML 报告 | 方便分享 |
| P2 | 增量扫描 | 大项目性能 |
| P2 | 更多检测器 | 扩展覆盖 |
